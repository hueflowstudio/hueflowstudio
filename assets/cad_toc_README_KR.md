# CAD 도면 리스트 반자동 생성 리습

AutoCAD에서 반복되는 시트폼 블록, TABLE 객체, 또는 같은 크기로 반복되는 도곽/폼 폴리라인을 기준으로 도면 리스트를 자동 생성하는 AutoLISP입니다.

기존 도면을 전부 속성 블록으로 바꾸지 않아도, 이미 작성된 도면의 시트폼 안에 있는 일반 텍스트나 TABLE 셀 값을 읽어 `SHEET / DWG NO / TITLE / SCALE` 목록을 생성합니다.

## 빠른 사용법

```text
APPLOAD로 cad_toc_auto.lsp 불러오기
→ TSET으로 기준 폼 1개 선택
→ 오른쪽용/아래쪽용처럼 다른 폼 종류가 있으면 TADD로 추가 등록
→ 번호/제목/스케일 위치 지정
→ DWG 저장
→ TLIST 실행
→ 도면 리스트 자동 생성
```

```text
[Hueflow CAD 도면 리스트 자동화 리습 사용법]

1. AutoCAD에서 APPLOAD 입력
2. cad_toc_auto.lsp 파일 불러오기
   - AutoCAD 2014에서 안 되면 cad_toc_auto_2014.lsp 호환판을 불러오기
3. 명령어 TSET 입력
4. 기준이 되는 시트폼/도곽 블록, TABLE 객체, 또는 도곽선 폴리라인 1개 선택
5. 화면 안내에 따라 SHEET NO, 도면번호, 도면제목, SCALE 위치를 각각 지정
6. 오른쪽용/아래쪽용처럼 폼 종류가 더 있으면 TADD로 다른 폼도 같은 방식으로 추가 등록
7. DWG 저장
8. 명령어 TLIST 입력
9. 도면 안의 같은 시트폼들을 자동으로 읽어서 도면 리스트 생성

주요 명령어

TSET / TOCFORMSET
- 첫 번째 기준 시트폼을 등록하는 명령어
- 오른쪽 제목란용, 아래쪽 제목란용처럼 폼 종류가 여러 개면 먼저 한 종류를 등록함

TADD / TOCFORMADD
- 다른 위치나 방향의 시트폼을 추가 등록하는 명령어
- 같은 DWG 안에 오른쪽용 폼과 아래쪽용 폼이 같이 있을 때 사용

TSTAT / TOCFORMSTATUS
- 현재 도면에 저장된 폼 세팅이 있는지 확인하는 명령어
- 데스크탑/다른 PC에서 안 될 때 먼저 확인하면 좋음

TLIST / TOCFORMSCAN
- 등록된 시트폼을 기준으로 전체 도면 리스트를 자동 생성하는 명령어
- 같은 블록/폼 안의 텍스트를 읽어서 표 형태로 정리함

TCFG / TOCCFG
- 도면번호 패턴, 스케일 인식 등 기본 설정을 조정하는 명령어

추천 사용 흐름

APPLOAD → cad_toc_auto.lsp 불러오기
→ TSET으로 첫 폼 종류 세팅
→ 다른 폼 종류가 있으면 TADD로 추가 세팅
→ DWG 저장
→ TLIST로 전체 리스트 생성
→ 빠진 항목이나 상세도는 수동 보정

주의사항

- 같은 시트폼/도곽 블록, TABLE 객체, 또는 같은 크기의 도곽선 폴리라인을 반복해서 쓰는 도면일수록 정확도가 높음
- 도면 제목, 도면번호, 스케일이 일반 텍스트 또는 TABLE 셀 값으로 들어있어야 인식 가능
- PC를 옮길 때는 TSET 후 DWG를 저장해야 폼 위치 세팅이 같이 넘어감
- 다른 시트에 있는 상세도나 폼 구조가 완전히 다른 도면은 수동 보정 필요
```

## 핵심 기능

- 반복되는 시트폼 블록, TABLE 객체, 또는 도곽선 폴리라인을 기준으로 값 칸 위치를 학습
- 같은 폼 전체를 자동 스캔
- 같은 블록 이름의 폼은 크기가 달라도 가로세로 비율이 같으면 함께 스캔
- 시트번호, 도면번호, 도면명, 축척을 표로 생성
- AutoCAD Table 또는 기존 목차선 위 TEXT 출력 지원
- `L-000` 같은 예외 항목은 수동 추가 가능

## 리습 파일

`cad_toc_auto.lsp`


## 기본 사용 순서

### 1. 폼 기준 설정

명령어:

```text
TSET
```

긴 명령어는 `TOCFORMSET`입니다. 짧게는 `TSET`만 입력하면 됩니다.

순서:

```text
1. 반복되는 시트폼 블록, TABLE 객체, 또는 도곽선 폴리라인 하나 선택
2. SHEET NO 값 칸을 박스로 지정
3. DWG NO 값 칸을 박스로 지정
4. TITLE 값 칸을 박스로 지정
5. SCALE 값 칸을 박스로 지정
6. DWG 저장
```

주의: 2~5번은 폼 전체가 아니라 값이 들어가는 칸만 작게 지정합니다.

`TSET`으로 잡은 폼 세팅은 현재 DWG 안에 저장됩니다. 다른 PC에서 같은 DWG를 열면 `APPLOAD` 후 `TLIST`로 바로 사용할 수 있습니다.

저장된 폼 세팅이 있는지 확인하려면:

```text
TSTAT
```

긴 명령어는 `TOCFORMSTATUS`입니다.

### 2. 리스트 생성

명령어:

```text
TLIST
```

긴 명령어는 `TOCFORMSCAN`입니다.

추천 선택:

```text
Create output? → Yes
Output mode → Table
Table insertion point → 빈 곳 클릭
```

## 보조 명령

```text
TOCSEMI
```

선택한 도면 제목란/텍스트 영역을 반자동 인식합니다.

```text
TOCSEMIALL
```

DWG 전체를 스캔하되, 기존 목차 영역을 제외하고 반자동 인식합니다.

```text
TOCCFG
```

도면번호 패턴, 축척 패턴, 출력 기본값 등을 설정합니다.

짧은 명령어는 `TCFG`입니다.

## 적합한 도면 조건

- 같은 이름의 시트폼 블록, 같은 크기의 TABLE 객체, 또는 같은 크기의 도곽선 폴리라인을 여러 도면에 반복 사용
- 블록 폼은 같은 이름이면 축소/확대되어 있어도 상대 위치 기준으로 읽음
- 도면번호, 도면명, 축척이 폼 안의 일정한 칸에 위치
- 값이 일반 TEXT/MTEXT 또는 TABLE 셀 값이어도 가능
- 폼이 스케일되어도 상대 위치 기준으로 추적 가능

## 한계

- 폼 블록 이름이나 TABLE 크기가 여러 종류면 첫 종류는 `TSET`, 추가 종류는 `TADD`로 등록합니다.
- 표지처럼 다른 폼을 쓰는 항목은 수동 추가가 더 빠를 수 있습니다.
- 도면명 텍스트가 칸 밖에 있거나 여러 객체로 너무 흩어져 있으면 일부 수정이 필요할 수 있습니다.

## 안 될 때 확인

### 한 장만 읽힐 때

- 기준 폼을 도면 안에서 반복되는 시트폼 블록으로 선택했는지 확인합니다.
- 같은 블록 이름의 폼은 크기가 달라도 읽지만, 블록 이름이나 방향이 다르면 `TADD`로 추가 등록합니다.
- 이미 만들어진 기존 리스트 표를 기준 폼으로 선택하지 말고, 실제 도면 시트폼/도곽을 선택합니다.

### 값이 엇갈릴 때

- `TSET`에서 SHEET NO, 도면번호, 도면제목, SCALE 순서대로 값 칸만 작게 감싸서 다시 지정합니다.
- 라벨과 값이 같은 칸에 있어도 읽을 수 있지만, 여러 칸을 크게 감싸면 옆 칸 글자가 섞일 수 있습니다.
- 도면명 글자가 칸 밖에 있거나 여러 객체로 쪼개져 있으면 생성 후 일부 수동 보정이 필요할 수 있습니다.

### 누락되는 도면이 있을 때

- 표지, 상세도, 별도 폼처럼 시트폼 구조가 다른 도면은 별도 `TSET`이 필요할 수 있습니다.
- 폼 안의 값이 일반 TEXT/MTEXT, 블록 내부 TEXT, ATTRIB, TABLE 셀로 들어있어야 읽을 수 있습니다.
- 빈 폼이나 도면번호가 없는 폼은 자동 리스트에서 빠질 수 있습니다.

### AutoCAD 2014에서 명령어가 안 먹을 때

- AutoCAD LT 2014는 AutoLISP를 지원하지 않아 `APPLOAD` 자체가 안 될 수 있습니다. 일반 AutoCAD 2014인지 먼저 확인합니다.
- ZIP 안에서 바로 불러오지 말고 압축을 푼 뒤 `cad_toc_auto_2014.lsp` 또는 `cad_toc_auto.lsp` 파일을 `APPLOAD`로 불러옵니다.
- `APPLOAD` 후 명령창에 `Loaded cad_toc_auto.lsp` 문구가 떠야 정상입니다. 이 문구가 없으면 리습이 로드되지 않은 상태입니다.
- 보안 경고나 로드 차단이 뜨면 `SECURELOAD` 값을 `0`으로 바꾼 뒤 다시 `APPLOAD` 해봅니다.
- `TSET`/`TLIST`가 `Unknown command`로 나오면 리습 로드가 실패한 것이므로 `APPLOAD` 단계부터 다시 확인합니다.
- TABLE 생성이 실패하는 구버전 환경에서는 자동으로 TEXT 출력 방식으로 대체될 수 있습니다.

## 권장 워크플로우

```text
TSET
→ DWG 저장
→ TLIST
→ 생성된 Table 확인
→ 누락/예외 항목만 수동 보정
```

## 쉬운 명령어 요약

```text
TSET = 폼 기준 설정
TADD = 다른 폼 종류 추가
TLIST = 도면 리스트 생성
TSTAT = 현재 폼 설정 확인
TCFG = 설정값 조정
```
